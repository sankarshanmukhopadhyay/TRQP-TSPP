# v0.10.0

Supply Chain Integrity Controls and Schema Contract — Release Set B.

## Included

- **`harness/tests/test_11_sci_controls.py`** — Supply chain integrity test
  module with three requirement-tagged tests:
  - `TSPP-SCI-01`: SBOM reference declared in metadata and URI resolves.
  - `TSPP-SCI-02`: Release provenance attestation declared and URI resolves.
  - `TSPP-SCI-03`: Signing key freshness declared (rotation policy or JWKS);
    enforces ≤ 365-day rotation cadence at AL3+.
  All three tests are **advisory** (skipped with an informational message) at
  AL1/AL2 and **required** at AL3/AL4. Override via `TSPP_EXPECT_AL`.

- **`schemas/CONTRACT.md`** — Schema stability contract declaring Stable,
  Extensible, and Experimental tiers for every schema in `schemas/` and
  `harness/schemas/`. Includes per-schema field tables and the AL contract
  pin upgrade checklist.

- **`docs/standards-alignment.md`** — Extended with TSPP-SCI-01/02/03 and the
  full AL3/AL4 requirement set (TSPP-AL3-01–04, TSPP-AL4-02–05) mapped to
  OWASP API Security Top 10, NIST SP 800-53 control families, and ISO/IEC 27002
  clauses.
