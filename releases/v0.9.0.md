# v0.9.0

Deterministic Report Metrics and Bridge Fixture Promotion — Release Set A.

## Included

- **Posture/coverage metrics in every report**: `conftest.py` now computes and emits `posture_score`, `coverage_index`, and `control_satisfaction` in the TSPP conformance report summary on every pytest run. These fields were present in the golden flow sample but were not actually generated by the harness — closing a schema-tool alignment gap that would have caused silent `generate-manifest.py` ingestion failures.
- **Bridge fixture schema**: `harness/schemas/tspp-bridge-fixtures.schema.json` validates the structure of `bridge_golden_fixtures.json`. The schema is checked at `pytest` session start via a `pytest_sessionstart` hook, preventing silent fixture drift.
- **Bridge fixtures promoted to CI**: `test_07_bridge_equivalence.py` now uses `harness/fixtures/bridge_golden_fixtures.json` as the default fixture path without requiring `TSPP_BRIDGE_FIXTURES` to be set. Bridge semantic equivalence now runs in every CI matrix cell.
- **Canonical fixture format**: `bridge_golden_fixtures.json` migrated to include `query` and `expected` canonical fields alongside the legacy `trqp_query`/`expected_trqp_outcome` keys. Both shapes are accepted for backwards compatibility.
- **`test_10_recognition_security.py` moved**: relocated from the repository root to `harness/tests/` where it belongs. Fixes `python -m compileall` scope and removes adopter confusion.
- **Updated conformance report schema**: `tspp-conformance-report.schema.json` now includes `posture_score`, `coverage_index`, and `control_satisfaction` as optional summary fields.
- **CI: stable operational metadata**: `TSPP_RUN_ID` and `TSPP_TARGET_ID` are now set from CI environment variables so every conformance report carries a stable, cross-referenceable run identity without post-processing.
