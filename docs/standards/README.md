# At-scale standards (secure distributed services)

These references are **informative** guidance for operators running TRQP endpoints as internet-exposed, distributed services.

They complement the control mappings in `../standards-alignment.md` by providing short “how to operationalize” notes per program/standard.

## Included references

- [NIST SP 800-207 (Zero Trust Architecture)](nist-sp-800-207-zero-trust.md)
- [OWASP API Security Top 10](owasp-api-security-top-10.md)
- [OpenTelemetry (service telemetry)](opentelemetry.md)
- [SLSA (supply chain levels for software artifacts)](slsa.md)
- [NIST SP 800-190 (container security)](nist-sp-800-190-containers.md)
- [CIS Kubernetes Benchmark](cis-kubernetes-benchmark.md)

## How to use this folder

1. Start with `docs/requirements.md` for normative requirements and control IDs.
2. Use `docs/standards-alignment.md` to map control IDs to your existing security program.
3. Use the notes in this folder to translate those mappings into **operator runbooks, evidence, and platform controls**.
